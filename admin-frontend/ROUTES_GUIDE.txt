SUNRISE SCHOOL CMS FRONTEND ROUTES GUIDE
========================================

Project type:
- React + Vite admin panel
- React Router based frontend navigation
- PHP backend is separate and already working


1. MAIN ROUTING FILE
--------------------

Frontend route definitions are in:
- src/App.jsx

This file is the central route map for the admin panel.
It connects URL paths to page components.

Router setup entry point:
- src/main.jsx

This file wraps the app with:
- BrowserRouter


2. MAIN LAYOUT
--------------

Shared admin layout is in:
- src/components/AdminLayout.jsx

This component provides:
- left sidebar
- navigation links
- main content area
- common admin shell layout

All page routes are rendered inside:
- <Outlet />

That means every page uses the same admin layout automatically.

5. FRONTEND SERVICES
--------------------

API service files are in:
- src/services/

Current files:
- src/services/api.js
- src/services/pages.js

src/services/api.js
Purpose:
- Creates shared axios instance
- Uses Vite environment variable:
  VITE_API_BASE_URL

Current .env:
- frontend/.env

Current backend base URL:
- http://localhost/sunrise-backend/api


src/services/pages.js
Purpose:
- Handles page-related API calls

Current functions:
- getPage(slug)
- updatePage(payload)

Used in:
- src/pages/HomePage.jsx


6. CURRENT LIVE FEATURE
-----------------------

The only module currently connected to real backend data is:
- HomePage.jsx

What it does:
- fetches the About page using slug: about-school
- displays page preview
- allows editing title and content
- saves updates back to backend

This is the current real working CRUD integration in the frontend.


7. SIDEBAR NAVIGATION
---------------------

Sidebar links are defined in:
- src/components/AdminLayout.jsx

If you want to:
- add a new frontend route
- show it in navigation

You usually need to update:
1. src/App.jsx
2. src/components/AdminLayout.jsx
3. create a page under src/pages/
